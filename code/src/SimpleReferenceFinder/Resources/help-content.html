Basic Settings

Search Type

First of all choose an appropriated Search Type. Available search types are:

Resolve references of files in other files.

The references of a list of input files is be resolved within the list of target file. The list of input files is determined by provided Source Patterns and the list of target files is determined by provided Target Patterns. Unfortunately, at the moment only the search type Resolve references of files in other files. is possible.

Base Folder

As next choose the Base Folder from where to start searching. The path must exist but can be selected by clicking button [...] next to the right of the edif box.

Search Patterns

Thereafter, provide a semicolon separated list of search patterns (for example, some file extensions with wildcards) as Source Patterns as well as Target Patterns. Standard wildcards such as * and ? are supported. Standard all files search patterns *.* are used if not search patterns are defined.

Search Options

Case Sensitive

Any string comparison to find fitting results is performed case sensitive if this option is switched ON. Otherwise string comparison takes place ignoring character casing.

Search Recursive

Searching for file takes place in each folder and sub–folder if this option is switched ON. Otherwise file searching is performed inside the Base Folder only.

Include Folder

This switch allows to enable or disable the inclusion of partial sub–folder names. An example should be useful to understand how it works.

Example

Consider folder structure below.

  • c:\base-folder
    • file-1.ext
    • file-2.ext
    • sub-folder-a
      • file-a1.ext
      • file-a2.ext
      • file-a3.ext
    • sub-folder-b
      • file-b1.ext
      • file-b2.ext

Scanning for affected source files returns this list of file names.

  • c:\base-folder\file-1.ext
  • c:\base-folder\file-2.ext
  • c:\base-folder\sub-folder-a\file-a1.ext
  • c:\base-folder\sub-folder-a\file-a2.ext
  • c:\base-folder\sub-folder-a\file-a3.ext
  • c:\base-folder\sub-folder-b\file-b1.ext
  • c:\base-folder\sub-folder-b\file-b2.ext

The analyze action takes each of the fully qualified file names from scanning result and tries to find current file name in each of the target files. As soon as the content analysis comes to file c:\base-folder\sub-folder-a\file-a2.ext, its Base Folder part c:\base-folder\ is removed which in turn results in sub-folder-a\file-a2.ext. And this string represents the actual search string!

Conclusion

The actual search string for the example above is sub-folder-a\file-a2.ext if option Include Folder is switched ON. In contrast to that, the actual search string for the example above is file-a2.ext if option Include Folder is switched OFF.

Change Separator

This switch allows to enable or disable the replacement of each path separator. In other words, the platform–specific path separator is replaced by its alternative path separator representation.

On Windows systems it means that the path separator '\' is replaced by '/'. On Linux systems it means that the path separator '/' is replaced by '/', witch is actually the same character.

Therefore, this switch is useful to find for example sub-folder/file-name.ext references.
